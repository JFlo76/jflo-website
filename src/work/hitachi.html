Overview

Hitachi is heavily investing in the IoT marketplace and the first step was to build Lumada, a platform for data visualization of industrial machinery.

I was brought in as a contractor at the design phase to help design and then build the UI for the software. Deliverables included sitemaps, userflows, wireframes, prototypes, hi-rez comps, styleguide/pattern library, and production-ready code.

Design Phase

Initially, the plan was to build a dashboard for quick insights and overviews, data aggregation, and geo-location for devices that are connected to the Lumada platform (see first image). From there, the user would be able to drill directly down to the device details, or navigate to the inventory pages (second image).

For the MVP release however, the main priority was to set up the ability to create and manage digital representations of devices (Asset Avatars) that send data from sensors to the Lumada platform. The dashboard feature is scheduled for the beta release.

Based on preliminary info from the PMs and engineers, the process for creating new Asset Avatars was coupled to the creation of an Asset Type (classification of device; e.g. "truck” or “printer") and whether the device needed a gateway to connect to the platform. Because of this, I suggested the user would need a guided pathway for successful creation of their inventory.

Later on, the engineers were able to remove some of the dependencies around these, so the UI changed to a self-guided type, where the user could choose what they wanted to create first. Since the user profiles for the MVP were going to be Hitachi engineers who would receive training on the product, we felt this was an appropriate decision.
